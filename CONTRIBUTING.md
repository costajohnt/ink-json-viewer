# Contributing

Thanks for your interest in contributing to ink-json-viewer!

## Getting Started

1. Fork the repository and clone your fork
2. Install dependencies: `npm install`
3. Create a feature branch: `git checkout -b my-feature`

## Development

- `npm run build` -- build the package
- `npm run dev` -- build in watch mode
- `npm test` -- run the test suite
- `npx tsc --noEmit` -- type-check without emitting
- `npx tsx examples/basic.tsx` -- run an example

## Making Changes

1. Write your code and add tests for new functionality
2. Make sure all tests pass: `npm test`
3. Make sure types are correct: `npx tsc --noEmit`
4. Commit your changes with a descriptive message
5. Push to your fork and open a pull request

## Pull Request Guidelines

- Keep PRs focused on a single change
- Include tests for new features or bug fixes
- Update documentation if the public API changes
- Reference any related issues in your PR description

## Reporting Issues

If you find a bug or have a feature request, please open an issue on GitHub. Include as much detail as you can: what you expected, what happened, and a minimal reproduction if possible.

## Code of Conduct

Be kind and respectful to other contributors. We're all here to build something useful together.
